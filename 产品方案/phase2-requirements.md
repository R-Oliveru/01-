# 第二阶段：需求开发页面 - 需求草案

## 1. 页面概述
**需求开发页面**是"01计划"的核心执行模块，负责将已采纳的需求转化为实际产品。该页面提供项目管理的全套功能，支持多需求并行开发。

## 2. 核心功能构思

### 2.1 项目看板视图
- **多需求并行显示**：每个需求作为一个独立项目卡片
- **状态列**：待开始 / 进行中 / 测试中 / 已完成
- **拖拽操作**：通过拖拽改变项目状态
- **进度可视化**：每个项目显示完成百分比

### 2.2 任务分解系统
- **层级化任务**：项目 → 阶段 → 任务 → 子任务
- **预设模板**：不同类型需求有不同的标准开发流程
  - **App开发流程**：需求分析 → UI设计 → 前端开发 → 后端开发 → 测试 → 发布
  - **网页开发流程**：原型设计 → 前端开发 → 后端集成 → 部署
  - **Agent开发流程**：Prompt设计 → 功能测试 → 集成部署
- **自定义流程**：支持创建个性化流程模板

### 2.3 资源与时间管理
- **时间估算**：为每个任务预估工时
- **实际耗时**：记录实际花费时间
- **资源分配**：标记任务负责人（个人使用时可标记"自己"或"待分配"）
- **截止日期**：设置里程碑时间点

### 2.4 技术栈与依赖管理（简化版）
- **智能推荐**：根据项目类型自动推荐技术栈组合
- **模板选择**：提供开发模板（如"快速原型"、"全栈应用"、"AI驱动项目"）
- **依赖可视化**：用图标和描述性语言展示技术选型，避免技术名词障碍
- **一键配置**：支持生成项目初始化命令或配置文件

## 4. 交互流程构思

```mermaid
graph TD
    A[从需求池选择项目] --> B[创建开发项目]
    B --> C[选择开发模板或自定义]
    C --> D[生成任务分解结构]
    D --> E[在看板中显示项目]
    E --> F{执行开发}
    F -->|完成任务| G[更新进度]
    F -->|遇到问题| H[添加备注/调整计划]
    G --> I[状态更新]
    H --> I
    I --> J{项目完成?}
    J -->|是| K[移至"已完成"状态]
    J -->|否| F
    K --> L[准备进入GTM阶段]
```

## 5. 关键设计决策（已确认）

### 5.1 项目与任务的粒度 ✅
- **选择**：粗粒度（阶段级别：设计、开发、测试等3-5个阶段）
- **说明**：符合vibe coding的快速迭代风格，避免过度分解带来的管理负担

### 5.2 进度跟踪方式 ✅
- **选择**：基于任务完成数量自动计算
- **公式**：进度 = 已完成任务数 / 总任务数 × 100%
- **说明**：自动计算减少手动维护，保持进度实时准确

### 5.3 技术栈管理 ✅
- **新方案**：智能推荐 + 模板化选择
- **实现方式**：
  1. **自动推荐**：根据项目类型（App/网页/小程序等）推荐常用技术栈
  2. **模板选择**：提供"快速原型"、"全栈应用"、"AI驱动"等预设模板
  3. **简化展示**：用图标+文字描述替代技术名词，如"🌐 前端框架"、"📱 移动端"
- **说明**：降低技术选择门槛，让vibe coding团队专注于需求实现

### 5.4 与第一阶段的数据衔接 ✅
- **选择**：一键转换，自动带入需求信息
- **流程**：选中需求 → 点击"开始开发" → 自动创建项目卡片
- **携带信息**：需求标题、描述、分类、标签、AI评审结果

## 6. 整体页面架构与导航

### 6.1 三阶段集成方案 ✅
- **首页默认**：打开应用时默认显示**第一阶段（需求收集）**
- **顶部导航栏**：固定显示三个阶段的切换标签
  ```
  [📝 需求收集]  [🛠️ 需求开发]  [🚀 GTM]
  ```
- **单页应用体验**：点击导航标签，页面内容平滑切换，无需刷新
- **状态同步**：三个阶段的数据实时同步，一处修改处处更新

### 6.2 页面布局调整
- **统一头部**：显示"01计划"Logo和用户信息（如当前项目数）
- **左侧导航**（可选）：快速跳转到具体项目或筛选视图
- **主要内容区**：根据当前阶段显示对应功能模块
- **底部状态栏**：显示数据统计（如总点子数、进行中项目数）

### 6.3 数据流设计
- **中央数据存储**：所有三个阶段的数据统一存储在IndexedDB
- **实时更新**：任一阶段的操作立即反映在其他阶段
- **项目状态流转**：需求 → 开发 → GTM 的完整生命周期跟踪
