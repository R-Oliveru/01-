# 第三阶段：GTM（Go-To-Market）需求草案

## 版本
v0.3 - 2026-06-11（更新：根据用户反馈，聚焦核心GTM手段/阶段记录）

## 概述
GTM阶段跟踪产品从开发完成到上线的全过程，包括市场推广、用户增长、盈利情况等。

## 核心决策确认

### 1. "累计用户数"定义 ✅ 已确认
- **指产品的累计用户数**（如"早睡计划"有100个用户）
- 平台本身用户数暂时为1（单用户模式）
- 引导页面展示：所有已上线产品的用户总数

### 2. GTM核心目标 ✅ 已确认
- **记录历史数据，复盘学习**
- **建立可复用的增长方法论流程**
- 让GTM过程更清晰，进度更可视化
- 督促团队按流程推进，提高执行效率

## 功能模块构思

### 模块1：上线准备检查表
| 检查项 | 说明 | 状态 |
|--------|------|------|
| 产品命名 | 是否确定最终名称 | □ |
| 域名/应用商店 | 是否注册 | □ |
| 基础页面 | 首页、关于页、隐私政策 | □ |
| 用户反馈渠道 | 客服邮箱、反馈表单 | □ |
| 数据分析 | 接入Google Analytics/友盟 | □ |

### 模块2：市场反馈与推广记录
- **推广渠道**：小红书、知乎、内网社区、Geo定位等
- **内容管理**：发布的文章/视频链接
- **市场反馈**：用户评价、评分、NPS、投诉建议
- **效果跟踪**：点击量、转化率、用户增长
- **成本记录**：推广费用、ROI计算

### 模块3：用户增长仪表盘
```
📈 用户增长曲线
├── 累计用户数：XXX
├── 日活跃用户：XXX
├── 用户留存率：XX%
└── 用户来源分布
```

### 模块4：盈利情况跟踪
- **收入来源**：广告、订阅、内购、赞助等
- **成本结构**：服务器、域名、推广费用
- **盈利状态**：亏损/持平/盈利
- **关键指标**：LTV、CAC、ROI

### 模块5：增长方法论库（可复用流程）
> 将验证有效的GTM手段固化为可复用的模板，形成增长方法论

| 模板名称 | 适用产品类型 | 核心步骤（固定+可自定义） | 预期效果 |
|----------|--------------|---------------------------|----------|
| **小红书冷启动** | 内容型产品 | 1.账号养号 2.内容规划 3.KOL合作 4.数据复盘 | 1000+初始用户 |
| **Geo定位推广** | 本地服务/工具 | 1.区域分析 2.本地渠道 3.地推活动 4.效果评估 | 精准本地用户 |
| **内网社区推广** | B端/内部工具 | 1.内部预热 2.帖子发布 3.同事反馈 4.迭代优化 | 内部用户采纳 |
| **自定义模板** | 任何类型 | [用户可添加/编辑步骤] | 灵活适应 |

**特点**：
- ✅ **固定+可自定义**：每个模板包含固定核心步骤 + 可添加自定义步骤
- ✅ **基于实际经验**：目前主要基于小红书实践，逐步积累其他方法
- ✅ **持续优化**：记录实际效果，不断改进模板
- ✅ **知识沉淀**：团队共享成功经验

## 数据模型思考

### 与第一、二阶段的数据关联
```
需求收集 → 需求开发 → GTM
 点子      项目       产品
   ↓         ↓         ↓
[创意] → [实现] → [上线]
```

### 需要记录的关键数据点
1. **产品基本信息**：名称、描述、分类、上线时间
2. **用户数据**：累计用户、活跃用户、留存率
3. **财务数据**：收入、成本、利润
4. **市场数据**：推广渠道、内容、效果
5. **里程碑**：重要版本更新、关键事件

## 关键决策确认 ✅

### 1. 跟踪指标 ✅ 已确认
- **用户增长**：日活跃用户、用户留存率
- **盈利情况**：收入、成本、ROI（投资回报率）
- **市场反馈**：用户评分、评论、NPS

### 2. 数据收集方式 ✅ 已确认
- **按需记录**：不强制每日记录，根据实际需要更新
- **手动输入**：通过结构化表单降低记录成本
- **不支持批量导入**：暂时保持简单，避免复杂化
- **无数据验证**：暂时不添加验证规则，信任用户输入
- **未来扩展**：可考虑自动同步（如接入Google Analytics API）

### 3. 核心GTM手段/阶段记录 ✅ 已确认
> 用户明确：暂时不需要定义"成功"标准，聚焦核心GTM手段/阶段的记录
- **记录GTM活动**：市场推广、渠道投放、内容发布等
- **跟踪阶段进展**：从上线准备到持续运营的关键节点
- **建立可复用流程**：将有效的增长手段固化为标准流程
